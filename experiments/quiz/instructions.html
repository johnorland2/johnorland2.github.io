Instructions

Overview:

Primarily, the application presents a sequence of interactive multiple choice questions. This interface could be used as part of a "trivia" game at a party. Alternatively, the interface could be adapted for usage as a quiz for study/memory aid purposes.

The end user may use mouse clicks to interact with the page. Clicks are used to transition between application phases, and to submit answers to questions.

About how to click on stuff properly:

Look out for orange rectangles that show a pointer (hand) cursor-icon when the cursor is moved into hovering vicinity of the element. The term "button" will be used to refer to an instance of these orange rectangle elements. A click of a button triggers some transition in the application. The button's textual label is a cue that suggests the behavior that's connected with a click. You might see other textual indications in the vicinity of these buttons, and these indications further elucidate what's going on.

Description of phases:

1. The first type of phase is the "menu". In this phase, the end user can see a "title" and a button that says, "play". Clicking the button transitions the application to a question phase.

2. The second type of phase is the "question" phase. The end user can see a textual indication of the question's prompt and the question's category. Below these indications, there are multiple clickable buttons. Each button represents a possible answer to the question; the button's textual label conveys a possible answer to the question. You get one chance to answer a particular question. If you click on one answer during a question phase, then your opportunity to answer that question is spent.

After clicking an answer button, the application is transitioned to a subphase that shows a textual indication that the answer was either correct or incorrect. This subphase also gives a button; the button either leads to another question phase, or leads to the summary phase if no further questions are available. Look for a textual cue that elucidates what's going on at this point.

3. The third (final) type of phase is the summary. In this phase, you can see a textual indication that explains how many questions were answered correctly.

Note about questions and subject matters:

The questions' prompts and answers were designed by the developer (John). They are inspired/motivated by hobby reading, recent events and similar things. The questions' prompts, answers (and connected facts, subjects, categories, etc) are not intended to be the main focus within this technical demonstration; they seemed sufficiently interesting to provide some intrigue, and to give motivation/reason for the existence of the application/interface. The application/interface (HTML, JavaScript, CSS, user-experience, etc) is intended as the main focus of the experiment. Further, the questions and answers might somehow misportray the subjects; this project is not authoritative. The questions and answer options involve the developer's opinion, and there might be some surprise/discordance with what's expected/imagined by the end user. The end user should take caution when deciding whether to try the questions. The end user may take interest in the topics and perform subsequent research as desired. The questions/answers are not meant to majorly influence the end user beyond the scope of brief interaction with this page; if influence is taken, it ought to be like "the application has reminded me about X subject, and I may find/verify information by consulting reference material" rather than "the application informs me to believe some information about subject X." Accordingly, a highly-detailed bibliography is not included.

The categories in this demonstration are: "anthropological", "computer programming", "driver's safety", "physical fitness" and "outdoors recreation".

A future version of the application could be reworked/adapted to have other categories, different types of questions and other modifications/enhancements.

Update (2/10/2024):

A previous version of the experiment used the word "trivia" in a questionable way. Whereas "trivia" might be defined as "facts (as about people or events) that are not well-known" in one context, it might elsewhere be defined as "unimportant matters". The former definition is the relevant one for this experiment because of the sample questions that were used. However the word "quiz" was provisionally swapped in, and replaced the word "trivia" to improve clarity. The code still could be used with either "quiz" questions or "trivia" questions for a different project. A future version of this experiment might use other questions to avoid unintended consequences (troublesome controversy, hurt feelings, etc), such as "arithmetic" questions; development of the "arithmetic" questions is a work-in-progress.
